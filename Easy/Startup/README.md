
# Startup - Abuse traditional vulnerabilities via untraditional means. 
Складність: Easy 

1. Розвідка (Reconnaissance & Enumeration)
Тут ти описуєш, як ти "мацав" машину.

  1.1. Сканування портів (Nmap): Твій скап з портами. Можеш додати коментар, чому ти вибрав саме ці прапори (-sC, -sV).

  1.2. Веб-розвідка:

Аналіз вихідного коду (чи були там коментарі розробників?).

Результати Gobuster або FFUF (приховані директорії).

Перевірка файлу robots.txt або субдоменів (якщо були).

2. Точка входу (Initial Access / Foothold)
Найважливіший момент: як ти перетворив інформацію на доступ.

2.1. Експлуатація вразливості: Опиши саму дірку (наприклад, LFI). Як виглядав фінальний Payload в URL.

2.2. Отримання реверс-шеллу: Який шелл використав (PHP, Python, Bash?) і як стабілізував термінал (python3 -c 'import pty; pty.spawn("/bin/bash")').

3. Підвищення привілеїв (Privilege Escalation)
Розбий це на два логічних етапи, бо це зазвичай різні техніки.

3.1. Горизонтальне переміщення (www-data -> User): Як ти знайшов дані користувача (паролі в бекапах, скриптах або конфігах).

3.2. Вертикальне підвищення (User -> Root): Твій шлях до "корони". Опис sudo -l, вразливих SUID-файлів або кривих скриптів.

Поради щодо оформлення блоків:
Використовуй "Blockquotes" для заміток:

Нотатка: На цьому етапі я застряг на годину, бо забув додати IP машини в /etc/hosts. Не будь як я — перевіряй хости!

Реверс-шелл: Краще виділи його в підпункт у розділі 2. Це технічний момент, який часто хочеться швидко скопіювати в майбутньому.

Результати: В кінці кожного розділу пиши, що ти отримав.
